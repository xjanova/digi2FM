The pitch

A modem in your pocket.

In the 90s, dial-up modems screeched files across phone lines. Digi2FM does the same thing — except the "wire" is the air between two phones, and the modem is software. Any file, any size, any format — encoded as audible (or near-ultrasonic) tones.

How it works

From bytes to tones, and back.

Digi2FM is a full software-defined modem stack. Here's the journey of a file from sender to receiver.

01

File → Binary

The selected file is read as raw bytes via expo-file-system and converted to a bit stream.

→
02

Chunk & Frame

Bytes are split into 64-byte packets with a sync word, header, sequence number, and CRC-16 checksum.

→
03

Error Correction

Optional Hamming(7,4) or 3× repetition coding adds redundancy so noisy environments don't kill the transfer.

→
04

FSK Modulation

Bit 1 → 1200 Hz, bit 0 → 2200 Hz. The Bell 202 standard, rendered as PCM samples at 44.1 kHz.

→
05

Speaker → Mic

Audio plays through the speaker. The receiver captures it via microphone and runs the Goertzel algorithm to detect tones in real time.

→
06

Decode & Verify

Bits are reassembled into packets. CRC check + sequence numbers ensure integrity. The file is written back to disk — pixel-perfect.

Features

Everything you need, nothing you don't.

📤

Send any file

Images, PDFs, audio, ZIPs — anything. The protocol is format-agnostic; it just sees bytes.

📥

Listen & receive

Open Receive mode, point your mic at the source, and watch packets arrive in real time.

📊

Real-time visualizer

Live spectrum analyzer + waveform display during both transmission and reception.

⚙️

Configurable modem

Tune baud rate, error correction mode, packet size, and tone frequencies to your environment.

🛡️

CRC-16 integrity

Every packet carries a CRC-16-CCITT checksum. Corrupt frames are flagged and re-requested.

🧮

Hamming(7,4) FEC

Forward error correction recovers single-bit flips per nibble, gracefully handling acoustic noise.

🔒

Optional encryption

TweetNaCl-powered authenticated encryption keeps your audio-transmitted secrets, secret.

📦

Packetized transfer

Files are chunked with sequence numbers — large transfers survive interruptions and resume cleanly.

📱

Cross-platform

Built on Expo + React Native. Ships to Android and iOS from a single TypeScript codebase.

FskModulator.ts 
// Generate one bit as a sine wave at the right frequency
function modulateBit(bit: number): Float32Array {
  const freq = bit === 1 ? MARK_HZ : SPACE_HZ;
  const samples = SAMPLE_RATE / BAUD_RATE;
  const buf = new Float32Array(samples);

  for (let i = 0; i < samples; i++) {
    buf[i] = Math.sin(2 * Math.PI * freq * i / SAMPLE_RATE);
  }
  return buf;
}

Performance

How fast does sound go?

Throughput scales with baud rate and inversely with the strength of your error correction. Here's the math, in real-world terms.

Baud FEC Effective speed 1 KB file 100 KB file
300 None ≈ 25 B/s ~40 s ~1h 7m
300 Repetition 3× ≈ 8 B/s ~2 min ~3h 20m
1200 None ≈ 100 B/s ~10 s ~17 min
1200 Repetition 3× ≈ 33 B/s ~30 s ~50 min

Tip: Use the lowest baud rate and strongest FEC in noisy rooms; crank everything up when phones are touching and the room is quiet.

Why?

When the internet isn't an option.

①

Air-gapped transfers

Move data between devices that should never share a network. Audio is a one-way physical channel by design.

②

Offline sharing

No cell signal, no WiFi, no cable. As long as both phones can hear each other, you can share.

③

Education & demos

A live, audible demonstration of modulation, framing, error correction, and decoding — all in one app.

④

Pure nostalgia

Let your phone screech like a 56k modem. Because we can, and because it sounds amazing.
